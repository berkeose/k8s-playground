# Kubernetes Learning Roadmap

## Goal

Build a strong practical foundation in Kubernetes and document the full learning process in a clean GitHub repository.

## Phase 1 — Fundamentals

Topics:

- What is Kubernetes?
- Why Kubernetes is needed
- Containers vs Pods
- Cluster architecture
- Control plane
- Worker nodes
- Basic kubectl usage
- YAML basics for Kubernetes

Outcome:

- understand the core building blocks
- create the first Pod manually
- inspect resources with kubectl

## Phase 2 — Core Workloads and Networking

Topics:

- Pods
- Labels and Selectors
- ReplicaSets
- Deployments
- Services
- Namespaces

Outcome:

- deploy and manage applications properly
- expose applications internally and externally
- understand how Kubernetes connects resources together

## Phase 3 — Configuration and Storage

Topics:

- ConfigMaps
- Secrets
- Volumes
- Persistent Volumes
- Persistent Volume Claims

Outcome:

- separate configuration from application images
- persist application data
- manage more realistic workloads

## Phase 4 — Production-Oriented Topics

Topics:

- Probes
- Resource requests and limits
- Rolling updates
- Ingress
- Helm
- Monitoring basics

Outcome:

- understand how Kubernetes is used more realistically
- prepare for more advanced projects and real environments

## Documentation Standard for Each Topic

For every topic, I want to include:

1. short concept explanation
2. important commands
3. YAML examples
4. what I personally understood
5. lab notes / troubleshooting

## Portfolio Goal

By the end of this project, this repository should show:

- clear understanding of Kubernetes fundamentals
- hands-on usage of kubectl
- ability to write and read Kubernetes YAML files
- structured technical documentation
- a visible progression from beginner concepts to practical usage
